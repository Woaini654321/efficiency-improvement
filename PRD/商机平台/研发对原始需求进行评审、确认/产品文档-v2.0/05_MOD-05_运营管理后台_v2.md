## 二、功能描述与页面清单

运营管理后台是平台治理中枢，为运营管理员（ROLE-03）提供**用户权限管理、数据隔离配置、内容审核、分类体系维护、SLA 时效监控、运营数据看板、公告发布、产品线维护**等全平台管控能力；所有运营写操作统一写入操作审计日志（查看入口见 MOD-06 PC-14）。

**覆盖 FEAT**：FEAT-0501~0507（7 项，见 §三）。**公告管理与产品线维护不新增 FEAT 编号**：按 `00_项目总纲_v2.md` §3.2，公告发布并入 FEAT-0503 内容治理域、产品线维护为 D3 新增运营页，均作为 MOD-05 子功能纳入，以维持全系统 **42 FEAT** 口径（详见 §2.1 注）。
**本期覆盖页面（7 页）**：
- PC-09 用户与权限管理（含数据隔离配置）
- PC-10 内容审核管理
- PC-11 分类体系维护
- PC-12 运营数据看板（Dashboard）
- PC-13 SLA 时效监控
- PC-24 公告发布（运营端）+ 公告列表（**新增/原 PC-15 重编号**）
- PC-26 产品线维护（**新增**）
**关联用户端页面**：PC-25 公告详情（归 MOD-03/用户端，本模块公告发布后推送触达）
**下期 Backlog（本文档仅登记）**：PC-23 批量发布任务、PC-21 会议任务管理
**跨模块引用**：PC-14 操作日志（MOD-06）

**保存流程（"保存并验证"）**：

1. 点击"保存并验证" → 系统对当前配置生成**配置快照**（snapshot，含时间戳、操作人、规则全量，供人工回退依据）。
2. 系统运行**预设越权用例集**（平台预定义的跨部门访问测试用例集合，覆盖 E-5 典型越权场景）→ 逐条产出 pass/fail。
3. 弹出 PC-09.CONFIRM-01 测试报告弹窗，展示每条用例结果 + 汇总（通过 N / 失败 M）。
4. **人工确认后生效**：
   - 全部通过 → 管理员点击"确认生效" → 配置生效 → Toast「配置已生效」→ 写操作审计。
   - 存在失败 → 弹窗高亮失败用例 + 二次确认；管理员可选择"返回修改"（配置不生效）或"知悉风险并强制生效"（需二次确认，理由必填，写审计告警）。**系统不自动回滚**；如需回退，管理员依据配置快照手动恢复（提供"回退到上一快照"操作）。

- 生效接口：`PUT /admin/data-isolation`（携带 confirm_token）；快照查询：`GET /admin/data-isolation/snapshots`；回退：`POST /admin/data-isolation/rollback`（body: snapshot_id）。

#### 24.3 发布后推送（→ 用户端 PC-25 公告详情）

- 发布（含定时到点）后：

| 规则编号 | 规则描述 |

| BR-072 | **撤回**：已发布公告可撤回（published → withdrawn），撤回后用户端入口/横幅下线，已生成通知不追回但详情提示已撤回。 |
| BR-073 | **已发布不可改正文**：published 状态仅可撤回；如需修改须撤回后新建或改草稿重发。 |

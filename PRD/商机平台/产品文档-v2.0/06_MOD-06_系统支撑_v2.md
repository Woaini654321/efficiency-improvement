## 2. 功能描述

系统支撑模块为平台提供基础设施能力：

- **企业 SSO 统一登录（FEAT-0601）**：框架 `quectel-code-security-starter` 提供 UAA/OAuth2.0 认证（STATELESS 纯 token，本地不存密码/不验签），前端用框架 `q-login-page`（账密 RSA 提交 UAA + OAuth 回调）；业务侧零登录代码，`MeController` 仅取当前用户；登录事件写审计日志。
- **全文搜索索引维护（FEAT-0602）**：基于 Elasticsearch/OpenSearch，内容发布/修改/下架/删除时经 RabbitMQ 异步（outbox）更新索引，异常降级 DB LIKE + 告警。
- **操作日志审计（FEAT-0603）**：write-only 防篡改，保留 365 天，敏感操作高亮，支持带上限导出（PC-14）。
- **中英文国际化（FEAT-0604）**：静态 UI / 枚举 / 分类名 / 通知模板多语言；UGC 不翻译；偏好写入 User.language。

**覆盖 FEAT**：FEAT-0601~0604（4 项）
**覆盖页面**：PC-00（登录页）、PC-14（操作日志）+ 系统后台能力（搜索索引维护、语言切换）

### 3.4 登录认证设计（FEAT-0601 核心 · 框架企业 SSO/UAA）

> 二次决策 TC-01（A2）落定：登录采用**框架默认企业 SSO（UAA/OAuth2.0）**，由 `quectel-code-security-starter` 提供，**不再自建本地账密库**。

**认证机制**：
- **无 Session、纯 token（STATELESS）**：后端不保存登录态。
- `TokenValidationFilter` 从请求 `Authorization` 头取 token → 回调 `{gateway-url}/uaa/current` 由 **UAA 远程校验**（平台本地不验签、不存密码）；校验通过后放行，角色统一加 `ROLE_` 前缀注入安全上下文。
- 前端 `q-login-page` 提供两条等价路径，均由 UAA 完成认证：①账号密码登录（密码 **RSA 加密后提交 UAA**）；②OAuth 回调（企业 SSO）。
- 业务侧**零登录代码**：`MeController` 仅通过 `/me`、`/me/id` 读取当前已认证用户。

**边界（非"降级"）**：账号密码同样走 UAA，**不存在"SSO 不可用降级到本地账密"**。真实边界为 **UAA 不可达 → 无法登录**，返回 401 `Authentication service unavailable`，归入依赖可用性/容灾（01 手册 §8），非降级到本地库。

**gateway-url 各环境配置**（切环境仅改一行）：

| 环境 | gateway-url |
|------|-------------|
| 生产 | `openapi.quectel.com/api/uaa` |
| 测试 | `fat-softweb.quec.com/api/uaa` |
| 开发 | `192.168.10.27` |

> 📝 密码复杂度、错误锁定、初始密码下发、密码重置等策略**全部由 UAA 负责，平台不实现**。

### 3.5 业务规则

| 规则编号 | 规则描述 |
|---------|---------|
| BR-049 | **框架 SSO/UAA 认证**：登录经 `quectel-code-security-starter` 走 UAA/OAuth2.0（STATELESS 纯 token）；`q-login-page` 的账密与 OAuth 两条路径均提交 UAA 认证，平台不自建账密库。 |
| BR-050 | **密码与锁定策略由 UAA 负责**：密码复杂度、连续错误锁定、初始密码下发、重置流程等**全部由 UAA 实现，平台不落地**（原自建 5 次锁定 30 分钟规则删除）。 |
| BR-051 | **token 由 UAA 校验/管理**：`TokenValidationFilter` 每次回调 `{gateway-url}/uaa/current` 远程校验；token 有效期与续期由 UAA 管理，失效后回登录页提示"登录已过期，请重新登录"。 |
| BR-052 | **凭据安全**：账号密码由 `q-login-page` **RSA 加密后提交 UAA**，平台本地不存储/不哈希/不校验密码；密码明文严禁写入日志/审计/异常栈。 |
| BR-053 | **登录审计**：SSO 登录与账密登录（成功/失败）均写 AuditLog `action_type=login`，记录操作人、结果、IP、User-Agent；失败标红。 |
| BR-054 | **账号状态门禁**：账号停用由 UAA 侧鉴权拦截；平台侧如命中本地维护的禁用标记同样阻断，提示联系管理员。 |
| BR-055 | **UAA 不可达处理**：UAA 远程校验不可达时返回 401 `Authentication service unavailable`，无法登录（依赖可用性，非降级到本地账密）。 |

### 3.6 异常处理

| 场景 | 处理方式 |
|------|---------|
| UAA 不可达（远程校验失败/超时） | 阻断登录 + 401 "Authentication service unavailable"，提示"认证服务暂不可用，请稍后重试或联系管理员"（非降级到本地账密） |
| OAuth 回调 token 校验失败 | 阻断登录 + "登录校验失败，请重试或联系管理员"；写审计 login 失败 |
| 账号未激活/停用 | 阻断登录 + "您的账号未激活/已停用，请联系管理员"（由 UAA 鉴权返回） |
| 账号或密码错误 | UAA 返回认证失败 → 提示"用户名或密码错误"（不区分具体项，防枚举）；错误锁定由 UAA 处理 |
| 网络/接口超时（10s，01 手册 §7） | "网络异常，请稍后重试" + 重试按钮 |

## 5. 系统功能

### 5.1 FEAT-0602 全文搜索与索引维护

> 目标：内容发布后可被快速检索；索引更新异步、最终一致；引擎异常可降级。

**引擎选型与架构**

| 维度 | 说明 |
|------|------|
| 搜索引擎 | **框架 `quectel-code-elasticsearch-starter` + `EsHelper`**（提供分页/高亮/聚合能力）；索引 mapping 与分词器由架构细化，中文分词建议 IK / smartcn，英文标准分词。**初期数据量小可先用 MySQL，上线前切 ES**（二次决策 TC-02） |
| 索引对象 | `Opportunity` 与 `OpportunityRequest` |
| 索引字段（全文） | `title` + `summary` + `content`（富文本正文取纯文本后入索引）；附带过滤字段：type、status、department_id、category、publisher_id、urgency、created_at 等用于筛选/排序 |
| 触发时机 | 内容 **发布 / 修改 / 下架 / 删除** 时更新索引（下架/删除 → 从可搜结果移除或置不可见标记） |
| 同步方式 | **RabbitMQ 异步 + Outbox 模式**：业务写事务内写 outbox 表 → 投递 MQ → 消费者更新 ES 索引，保证业务库与索引**最终一致**（不阻塞发布主流程） |
| 一致性口径 | **发布后 30s 内可搜**（正常路径通常秒级；30s 为 outbox → MQ → 索引链路的一致性上界，与 AC-408 对齐）。超 30s 未可搜视为异常，触发告警 |
| 搜索响应 | ≤ 1s（P99，01 手册 §7）；关键词最小长度 2 字符 |
| 相关度与排序 | 默认按**相关度**（引擎 BM25/评分，title 权重 > summary > content）；用户可切换 `sort_by`（如 created_at、view_count、like_count）+ `order`（asc/desc），排序参数统一走 API 契约（01 手册 §API 契约）。默认 `sort_by=relevance, order=desc` |
| 降级方案 | 引擎不可用/超时（30s）→ 降级为**数据库 LIKE 模糊查询**（title/summary，content 可选，性能受限），前台 Toast 无感或轻提示，**管理后台告警**（引用 01 手册 §8 外部系统集成与降级） |
| 权限过滤 | 搜索结果需叠加可见范围与部门隔离（D4，交集取严）；下架/删除内容不出现在普通用户结果中 |

**搜索索引维护业务规则**

| 规则编号 | 规则描述 |
|---------|---------|
| BR-056 | **异步最终一致**：索引更新经 RabbitMQ 异步 outbox 完成，绝不阻塞内容发布/修改主流程；MQ 消费失败按重试策略重投，多次失败记录告警。 |
| BR-057 | **降级告警**：搜索引擎异常 → 自动降级 DB LIKE，并在管理后台产生告警（可查看），恢复后自动切回引擎。 |
| BR-058 | **索引与权限一致**：搜索结果必须经可见范围/部门隔离二次过滤，禁止越权命中（D4）。 |
| BR-065 | **删除/下架同步**：内容 delete → 索引删除；archive（下架）→ 索引标记不可见，重新上架后恢复可见。 |

### 5.2 FEAT-0604 中英文国际化（i18n）

> 决策要点（C-02）：**UGC 不翻译**。海外用户看**英文界面 + 原文内容**。偏好写入 `User.language`。
> 二次决策 TC-03：静态 UI/菜单/字段/枚举走框架 `vue-i18n` 双语（zh-CN/en-US）；**后端错误码/提示走框架 `MessageSource`**；分类名等运营维护内容按**决策 B** 用 Category 可选 `name_en`（不填回退中文）。

**多语言范围（分层方案）**

| 层次 | 内容 | 方案 | 是否翻译 |
|------|------|------|---------|
| L1 静态 UI 文案 | 菜单、按钮、表头、提示、空状态、字段、前端错误文案等 | 框架 `vue-i18n` 双语资源包（zh-CN / en-US），前端按当前语言渲染；后端错误码/提示走框架 `MessageSource` | ✅ 翻译 |
| L2 枚举显示名 | 状态、操作类型、通知类型、urgency 等所有 ENUM 的展示名 | 枚举字典维护中英双列显示名（见 §4.5 示例），按语言取列 | ✅ 翻译 |
| L3 用户维护的分类名 | 运营在平台维护的分类体系（Category）名称 | **决策 B**：Category 增加**可选字段 `name_en`**（PC-11 表单加"英文名称（可选）"）；运营不填时英文界面**回退显示中文名**。公告/字典类同理可选双语 | ✅ 翻译（运营维护，可选） |
| L4 通知/邮件/飞书模板 | 站内通知、邮件、飞书消息模板文案 | 模板按语言维护双版本，发送时按接收人 `User.language` 选模板（模板变量填充原始业务数据） | ✅ 翻译（模板层） |
| L5 UGC（用户生成内容） | 商机/需求**富文本正文**、方案正文、**评论/回复**、用户填写的自由文本 | **不翻译**：海外用户看英文界面框架 + 原文内容；不提供机器翻译、不做双语存储 | ❌ 不翻译 |

**实现与持久化**

| 维度 | 说明 |
|------|------|
| 触发方式 | APP-SHELL TOPBAR 语言切换（zh-CN / en-US），全角色可用 |
| 支持语言 | zh-CN（默认）、en-US |
| 持久化 | 登录用户：写入 `User.language`（跨设备生效）；同时写 localStorage（未登录/首屏快速渲染）。优先级：User.language > localStorage > 浏览器 Accept-Language > 默认 zh-CN |
| 切换效果 | 切换后 L1~L4 内容按新语言渲染；L5 UGC 保持原文不变 |
| 缺失回退 | L3 分类名 en 缺失时回退 zh 显示（可后台提示补全）；L1 缺失 key 回退 zh 并记录，避免出现 raw key |

**i18n 业务规则**

| 规则编号 | 规则描述 |
|---------|---------|
| BR-066 | **偏好持久化**：语言切换写入 User.language（登录态）+ localStorage；下次登录沿用 User.language。 |
| BR-067 | **UGC 不翻译**：富文本正文、方案、评论/回复等用户生成内容一律原文呈现，不做任何翻译或双语存储。 |
| BR-068 | **模板按接收人语言**：通知/邮件/飞书发送时按接收人 User.language 选模板；模板不存在对应语言时回退默认 zh-CN。 |
| BR-069 | **回退策略**：分类名/枚举/文案缺失目标语言时回退 zh-CN 并保证不出现 raw key。 |

## 6. 数据模型与 API 契约引用

> 本模块不重复定义实体与契约细节，统一引用 `01_全局规约手册_v2.md`。此处仅列出本模块涉及的关键项。

**涉及实体（引用 01 手册数据模型 ER）**

| 实体 | 用途 | 关键字段 |
|------|------|---------|
| `User` | 当前用户 / 语言偏好 | 登录身份来自 UAA（id/username/phoneNumber/email/name/tenantId/roles，**无本地 password 字段**）；平台本地扩展 `language`(enum: zh-CN/en-US)、部门/产品线归属等 |
| `AuditLog` | 操作日志 | id, operator, action_type(ENUM-ACTION-TYPE), target, target_type, target_id, result, ip, user_agent, before_snapshot, after_snapshot, created_at（write-only，365 天） |
| `Opportunity` / `OpportunityRequest` | 搜索索引对象 | title, summary, content + 过滤/排序字段 |

**涉及接口（命名与规范引用 01 手册 §API 契约：复数 kebab 资源、统一分页 `page`/`page_size`、排序 `sort_by`/`order`、鉴权头、统一错误体、写操作 Idempotency-Key）**

| 接口 | 方法/路径（示例，以 01 手册契约为准） | 说明 |
|------|--------------------------------------|------|
| 登录认证 | 框架 `q-login-page` → UAA | 账密（RSA）或 OAuth 均提交 UAA 认证；平台不提供本地 `/auth/login` |
| 当前用户 | `GET /me`、`GET /me/id` | `MeController` 返回当前已认证用户（业务侧零登录代码） |
| token 校验 | `TokenValidationFilter` → `{gateway-url}/uaa/current` | 每次请求远程校验（STATELESS） |
| 搜索 | `GET /search?keyword=&type=&sort_by=&order=&page=&page_size=` | 全文检索，权限过滤 + 相关度/排序 |
| 语言偏好更新 | `PATCH /me/language` | 写 User.language |
| 操作日志列表 | `GET /audit-logs?operator=&action_type=&date_from=&date_to=&page=&page_size=` | 仅 ROLE-03 |
| 操作日志详情 | `GET /audit-logs/{id}` | 含前后快照 |
| 操作日志导出 | `POST /audit-logs/export` | ≤ 10,000 条，IP 脱敏 |

## 三、v1 → v2 核心改动 10 条速览

1. **范围收敛（D1/H-1）**：v1.0 = 仅 6 模块 **42 FEAT** 核心闭环；讨论区/情报中心/工具助手/会议/我的任务/批量任务/吐槽墙等无规格功能群移出下期；仅保留公告（轻量运营）。
2. **SLA 单一模型（D2/H-2）**：按 urgency 自动计时（特急 2h/紧急 4h/普通 24h），删除"用户自设截止时间/deadline/DatePicker"；本期**只监控首响**；固定三级升级链 L1→L2→L3；进入"方案收集中"即 responded 停表停升级，倒计时仅 Pending 显示。
3. **组织结构建模（D3/H-3；TC-04/A2 修订）**：新增 Department/ProductLine/ProductLineMember 实体与组织模型章节；**不接 ORG/UPM，部门/产品线/用户归属均由运营在平台本地维护**（PC-26；UAA 仅 7 字段、无部门/工号）；SLA 升级人从本地映射解析（L1 产品线负责人/L2 产品管理部/L3 管理层）。
4. **可见性收口（D4/H-4）**：需求可见范围(all/dept/personnel) **∩** 部门隔离（取交集、更严生效）；商机全员可见（不加 visibility_scope）；废止 BR-013"对所有已登录用户可见"。
5. **权限矩阵单一真理源（D5/H-5）**：统一 **3 角色 × 42 功能**（废止"38 功能"，补 FEAT-0209~0212 与 ROLE-03 分支）；需求发布仅 ROLE-01；单人单角色，运营入口仅 ROLE-03；**"谁下架谁恢复"**消除下架死锁。
6. **关键指标机制（D6/H-7）**：北极星 = **需求响应闭环率**；采用**分阶段基线机制**（首月采基线、次月定目标）；五项过程指标同步采集，运营红线（发布量<20、触达率<80%）为初始经验值。
7. **页面重编号（H-8）**：消除一号两页——PC-15 仅"通知偏好设置"；公告发布→**PC-24**、公告详情→**PC-25**、产品线维护→**PC-26**；PC-09 补入运营中心导航；PC-16→PC-25。
8. **统一 API 契约（H-9）**：RESTful 复数 kebab 资源、状态变更用子资源动作、统一分页(page/page_size)/排序(sort_by/order)/鉴权头/错误体/幂等(Idempotency-Key)；文件上传走框架 `oss-starter`（`ossTemplate` 预签名，见 §4.1 TC-05，不再自建 `/files/upload`）。
9. **数据模型单一权威 ER（H-10）**：全主业务实体加 `version` 乐观锁；Interaction 唯一约束 + `is_deleted` + `parent_comment_id`；新增 ViewLog(去重)/NotificationBatch(batch_id)/Subscription；采纳以 `adopted_response_id` 为单一真相源；移除 `score_*` 孤儿字段。
10. **其他一致性收口**：评论最多 2 级 + 软删占位（D7）；强制确认改 ROLE-02 手动勾选、无自动判定（D8）；冷启动默认订阅本部门相关分类 + 同类 10 分钟合并（D9）；保留期审计 365/软删 180/通知 90；自动保存 60s；分页列表 20/卡片 12；附件白名单统一；穿透测试降级（去"自动回滚"承诺）。

### 4.1 技术选型（原待确认 → **已定**，基于 Quectel-code 框架能力 + 用户决策 A2/B/C）

> 二次决策 2026-07-18 落定，详见 `../决策纪要与修正基线.md` §五「二次决策：TC-01~05 技术选型落定」及 `06_MOD-06` §9。

| 编号 | 事项 | 最终方案 | 依据 |
|------|------|---------|------|
| TC-01 | 登录 / 账密库 | **已定**：框架默认企业 SSO（UAA/OAuth2.0，`security-starter`，STATELESS 纯 token，`TokenValidationFilter` 回调 `{gateway-url}/uaa/current` 远程校验）；**不自建本地账密库**，密码/锁定由 UAA 负责；前端 `q-login-page`（账密 RSA + OAuth 回调）；无"降级本地账密"，UAA 不可达即 401 | A2 |
| TC-02 | SSO 协议 / 搜索引擎 | **已定**：SSO 用框架 UAA/OAuth2.0；全文搜索用框架 `elasticsearch-starter`+`EsHelper`（发布/改/下架经 MQ 异步更新索引，异常降级 DB LIKE + 告警；初期可先 MySQL、上线前切 ES） | 框架能力 |
| TC-03 | i18n | **已定**：静态 UI/菜单/字段/枚举走 `vue-i18n` 双语、后端错误码走 `MessageSource`、UGC 不翻译；Category 增可选 `name_en`（**决策 B**，不填英文界面回退中文） | B |
| TC-04 | 组织数据 | **已定**：**不接入 ORG/UPM，保持单体**；UAA 仅 7 字段（无部门/工号）；**部门、产品线、用户归属、各级升级人均由运营在平台本地维护**（PC-26）；SLA 升级人从本地映射解析，不用 `getUserSupervisorIds` | A2 |
| TC-05 | 存储 / 审计 | **已定**：文件走框架 `oss-starter`（`ossTemplate` 预签名）+ 前端 `q-upload`/`q-file-list`（删自建 `/files/upload`）；审计字段 `mysql-starter` 自动填充；操作审计自建表+AOP；热数据 MySQL 365 天，超期归档 **ClickHouse**（`clickhouse-starter`）仍可查询（**决策 C**） | C |

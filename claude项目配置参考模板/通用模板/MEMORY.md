## 2. 项目强制工作流程（不可违背）

1. 新需求 / 功能 / 重构 → 先头脑风暴/方案澄清。
2. 确认方向 → 写实施计划 → 计划模式 → 等待用户批准。
3. 实施 → 新增/修 bug 按 TDD 先写失败测试；优先 Edit，禁止直接 Write 整文件；改前先 Read。
4. 完成前强制验证（前后端构建 + 前后端测试 + 全量回归 + 联调 + 安全项，全集见 `AGENTS.md` QA 门禁）。

6. **严禁**主动 `git commit` / `git push` / 包安装 等高危操作（全集见 `CLAUDE.md` 第 12 节）。

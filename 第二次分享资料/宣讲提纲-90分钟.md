# Claude Code 提效实战 · 宣讲提纲（90 分钟版）

> 用法：每页 = 一张幻灯片。**【讲】**是给讲师的口播要点，不上屏。
> 节奏设计：前 15 分钟抓住高层，中间 60 分钟给开发干货，最后 15 分钟落到行动。
> 全程原则：**每个论点配一个自己项目的真实数据**，不讲通用鸡汤。

---

## 开场（5 分钟）

### P1 标题页
**用 AI 把一个人变成一支队伍 —— Claude Code 项目实战与团队提效**

【讲】自我介绍 30 秒：15 年全栈。过去几个月用 Claude Code 做了 3 个 0→1 项目和 1 个存量系统提效，今天全部内容来自这四个项目的真实记录，没有一页是网上抄的。

### P2 今天回答四个问题
1. 到底能提多少效？（真实数据）
2. 每个岗位怎么用？（不只是写代码）
3. 团队怎么保证一致性和质量？
4. 公司怎么落地不翻车？

【讲】提醒高层重点看 1、4，开发重点看 2、3。文档已发，今天讲骨架。

## 第一部分：为什么（15 分钟）

### P3 三个真实数据点
| 项目 | 传统预估 | 实际 |
|---|---|---|
| 音频转换器 0→1 | 2~3 周 | **设计当天定稿，当天交付**（36 测试全绿） |
| web-tool 平台 | 前后端联调 1~2 周 | **两天 3 个 PR，联调零返工**（294 测试） |
| 工单系统审计 | 数周专项 | **1 天走查挖出 3 个 P0 安全漏洞** |

【讲】逐条展开 30 秒。强调第三条：JWT 伪造漏洞长期存在、一直没人发现——AI 不只是快，是能干人干不了的事。

### P4 价值四象限：提速只是表象
提速（编码快）/ 提质（挖出人漏掉的漏洞）/ 降门槛（前端写后端）/ **资产化（经验变成可继承的文件）**

【讲】重点讲资产化：员工经验第一次变成可迁移、可审计的显性资产（现场展示一个 CLAUDE.md 和 skill 文件）。这是组织级复利，离职不带走。

### P5 认知对齐：AI 是需要管理的高级工程师，不是自动售货机
人负责：定义问题、拍板方案、审查验收。AI 负责：探索、实现、测试、文档、给证据。

【讲】用错误期待的人两周就放弃，建立正确模型的人产出翻倍。这是培训存在的意义——工具大家都有，差距在用法。

## 第二部分：怎么用（30 分钟）

### P6 八个姿势纠错（针对"会用但不深"）
1 先澄清再动手 / 2 复杂任务先计划模式 / 3 一任务一会话 / 4 维护 CLAUDE.md / 5 完成要证据 / 6 大任务拆小步 / 7 跑偏立刻打断 / 8 不只当代码生成器

【讲】现场举手调查"谁用过计划模式"——通常不到 20%，这就是"会用但不深"的差距所在。挑 2、4、5 展开讲，其他指向文档。

### P7 核心工作流六步法（流程图：每个阶段后面都有一道评审门禁）
需求澄清 →⟨评审⟩→ 设计 spec →⟨多角色评审 1~3 轮⟩→ 计划 →⟨评审+人工批准⟩→ TDD 分块实施 →⟨逐块评审⟩→ 全量终审 → 验证收口

【讲】本页最重要的一句话：**多角色评审不是流程中的一步，而是每个阶段产物的收口门禁**——需求、设计、计划、每个代码块，产出即评审，must-fix 清零才放行，1~3 轮收口。"零返工"的真相是每一层的错误都被拦在本层。举思政项目例子：spec 轮抓到 ThreadLocal 竞态、plan 轮抓到契约约定问题——同一批问题流到编码后才发现，返工成本放大十倍。人工杠杆率最高的点仍是"批准计划"：审计划 5 分钟，胜过盯代码 5 小时。

### P8 提效技巧 Top3（必装）
① CLAUDE.md = AI 的入职手册（展示真实文件结构）
② 计划模式 = 把返工消灭在动手前
③ 上下文管理 = /clear /compact + settings.json deny 排除敏感/大文件（注意：网传的 .claudeignore 不是官方机制，文档 4.3 有纠错）

【讲】强调红线教训：思政项目允许提交凭据（内网），工单系统严禁明文密钥——**红线按项目写，不能复制粘贴**。顺带点一句 Token 经济学（文档 4.10）：省 token 和提质量是同一件事——上下文污染既烧钱又降质，"一任务一会话 + 结论落盘 + 跑偏立刻打断"三个习惯让每个 token 只为新信息付费，成本敏感的同事重点看这节。

### P9 七大岗位提效地图（矩阵图，每岗位一句话亮点)
PM：进度从"问人"变"问代码库" / 产品：4 套 HTML 样张用看的选 / 架构：契约先行零返工 / 前端：155 组 i18n 键一天补齐 / 后端：幂等与竞态专项审查 / 测试：8×8 穷举矩阵 / 安全：1 天挖 3 个 P0

【讲】本页的目的是让每个人找到"跟我有关"的入口。细节全在手册第 5 章，各岗位回去照抄提示词模板（附录 C）。

### P10 全流程不跑偏：单一事实源
PRD → spec → plan → 代码 → 评审 → 验收，全部版本化文档进 git，下游只认上游文档，决策写回文档。

【讲】举例：PM 拍板"发布≠完结"直接写入 spec 决策记录，从此没人再问第二遍。变更成本从"开会对齐"降到"文档 diff"。

### P11 事实沟通：观点会扯皮，事实不会
"做得差不多了" → "37 测试全绿，2 项待复跑"
"应该没影响" → "6 处调用，2 处受影响，已加回归测试"

【讲】展示工单系统走查文档截图：每条问题都有 file:line 证据+复现+修复建议+状态。一份文档同时喂给开发/测试/PM/高层，零翻译损耗。这就是沟通效率的本质提升。

## 第三部分：进阶——从一个助手到一支队伍（20 分钟）

### P12 Subagent：任务级分身（架构图）
主会话保持全局视野，子代理去做：大范围探索 / 独立任务块 / 多角色评审。核心收益：**上下文隔离**——垃圾细节不污染主判断。

【讲】强调最高性价比用法是多角色评审：6 个子代理各戴一顶帽子（PM/架构/前后端/测试/安全）审同一份产物，成本几分钟，相当于开了场评审会。

### P13 SDD 子代理驱动开发 + Agent Teams
SDD：每个任务块 = 实现代理+评审代理接力，主会话只调度终审（NCM 8 块当日交付 / 工单系统 11 任务一天闭环）
Agent Teams：平级 agent 长期共存各守一摊（前端+后端+测试并行）

【讲】给选型口诀：单任务用 subagent，大专项用 teams，前提都是"契约先行+终审兜底"。顺带讲反模式：小任务开团队=烧钱零收益；无契约并行=集成灾难。

### P14 Harness 自我进化：让环境越用越强（四级阶梯图）
记忆（记得）→ 规则 CLAUDE.md（必须遵守）→ 技能 Skill（跨项目继承）→ 钩子 Hook（强制拦截）

【讲】讲真实故事：我连续用选择题问需求被拒两次，沉淀成一条 feedback 记忆，从此再没犯过。第 1 个项目踩的坑，第 4 个项目第一天就免疫——**经验的边际复制成本为零**。现场展示 solo-fullstack-project skill 的 Lessons Learned 章节。收尾放彩蛋：这套进化资产全在 `~/.claude/` 一个文件夹里，换电脑拷走它，整个"进化后的环境"跟着走——我 Mac 换 Windows 就是这么恢复的，今天讲的案例数据全部来自那次迁移（文档 9.6）。

### P15 案例串讲：四个项目一张表
思政（流程即质量）/ baseProject（授权即速度）/ web-tool（契约即并行）/ 工单系统（认知即入场券）

【讲】重点讲工单系统——多数人日常是存量系统：入场三板斧（CLAUDE.md/AGENTS.md/MEMORY.md）当天完成当天开工，第二天六角色走查。这是每个人下周就能复制的路径。

## 第四部分：团队与治理（15 分钟）

### P16 一致性靠机制不靠自觉
共享配置随仓库分发（clone 即继承）/ CLAUDE.md 变更走 PR / Hook+CI 门禁自动拦截

【讲】新人 clone 仓库即自动继承全部规范和红线——新人培养从数周到半天。

### P17 质量：AI 代码过的关卡比人写的更多
分层验证金字塔：AI 自验（跑给我看）→ 自动化门禁 → 多角色评审 → 人工终审
三道防线防 AI 特有风险：真实样本验证 / 证据式汇报 / 计划冻结范围

【讲】讲 NCM 3.0 格式坑：自造测试全绿、真实文件乱码——"AI 的自测可能自洽但脱离现实，关键路径必须注入真实数据"。这是本次培训最值钱的一条经验。

### P18 安全红线（一页说清）
密钥绝不进上下文 / 生产环境零自动操作 / 自动审批类动作永远人按最后按钮 / 数据分级政策
反转认知：工单系统 3 个 P0 都是 AI 挖出来的存量漏洞——**配合红线，AI 是安全的净增益**。

【讲】建议把"AI 安全走查"纳入每次大版本发布流程——这是本次培训给安全团队的直接可执行建议。

### P19 度量与成本（一页纸）
先测基线再对比：交付周期/返工率/缺陷密度/资产沉淀数。
成本：订阅费 ≈ 人力成本 1~3%，每月省半天即回本。
⚠️ 不要考核"AI 使用率/生成占比"——会催生表演式使用。

【讲】承诺试点期后拿真实数据来汇报，不用行业吹的百分比。

### P20 落地路线图：试点 → 规范化 → 规模化
阶段 1：3~5 人种子队 + 真实项目 + 基线（1~2 月）→ 门禁：正收益+零安全事故
阶段 2：规范 v1 + 全员培训 + 结对（2~3 月）→ 门禁：80% 过基本功考核
阶段 3：全覆盖 + 月度度量汇报（持续）

【讲】向 CTO 要三样东西：试点授权、种子队员时间、以及一句"AI 提升杠杆不做减员工具"的公开表态（化解职业焦虑，直接决定推广成败）。

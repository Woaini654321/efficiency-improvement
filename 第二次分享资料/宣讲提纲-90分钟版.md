## 全场设计

**一句话主线**（开场 30 秒说一遍，结尾再说一遍）：

> 你用的不是一个工具，是一个智能体——只有模型人人相同，其余部件决定提效；部件可以养、可以抄、可以复制，我们用两个项目证明了这一点。

**三个记忆点**（听众散场只会记住三件事，主动替他们选好）：

1. **只有模型是一样的**（智能体观）
2. **AI 提供马力，门禁提供刹车，人掌方向盘**
3. **姿势可复制，不是天赋**（商机平台实验）

## 第 ① 幕：悬念开场（10 min）

**目标**：让所有人放下手机——用一个他们都见过的现象开场。

**开场白（30 秒逐字稿）**：

> "先问大家一个问题：同一个 Claude Code，为什么有人用两周就说'AI 不行'退回去手写，有人产出翻倍还越用越快？今天整场分享只回答这一个问题。答案一句话：你用的不是一个工具，是一个智能体——而且每个人手里的那个，都不一样。"

**讲述要点**（对应文档 1.4）：

1. 现场举手互动："用过 Claude Code 的举手？觉得不好用的别放下"——用真实分布引出 A/B 对比
2. 放 **开发者 A vs 开发者 B 对比表**：同一个软件、两个智能体，逐行念部件差异
3. 放 **七块板对比图**（柱状 vs 折线）：唯一持平的就是"模型"那一根
4. 抛出三条推论（提效在部件不在模型 / 智能体是养出来的 / 个人部件+团队共享层）

**过渡句**："部件怎么补？不用记 16 章，拿到项目的前三天做对，就补齐了一半。"

## 第 ② 幕：拿到项目怎么做（25 min）

**目标**：给出可带走的操作主线。只展开三个机制，其余报菜名。

**结构**（对应文档：开工路线图 + 第 3 章 + 8.2/8.3）：

| 小节 | 时长 | 讲法 |
|---|---|---|
| 开工路线图 | 5 min | 两条线各一张表：0→1（商机平台路线）/ 存量（工单路线）。强调共同点：**第一天都不写业务代码** |
| 机制一：六步法 + 评审门禁 | 8 min | 六步流程图一带而过，重点讲 **评审漏斗图**——"零返工的真相是层层拦截"，每层配一个真实拦截案例（登录方案纠错 / @EnableAsync / 前后端双缺 P0） |
| 机制二：黑盒纪律 | 6 min | 讲 env 白屏血泪故事（有情节、有反转，全场最好讲的段子）：开源直觉 → 白屏 404 → 原来框架 env 有白名单、登录本来就是企业 SSO。收口金句："**文档和直觉都可能骗你，字节码不会**" |
| 机制三：样板→镜像 | 6 min | 一个模块打穿 → 8 路子代理镜像 → 中心化门禁。两条铁律：子代理不跑门禁、共享文件不并行改 |

**明确不讲（报菜名，指向手册）**：Token 经济学（4.10）、模型分档（8.5）、记忆进化（第 9 章）、度量体系（第 14 章）——"手册里都有，回去查"。

**过渡句**："方法听着都对，问题是——真做得成吗？看证据。"

## 第 ③ 幕：证据（25 min）

### 商机平台（15 min，对应 10.1）——看图讲故事，不念表格

1. **实验设定**（1 页 PPT）："一个不熟悉公司环境的开发者 + 公司 skill + 一份 PRD，全程不写一行代码"——强调刻意苛刻
2. **四张里程碑截图逐张放**（每张 30 秒讲一个瞬间）：
   - 骨架完成 → "一句话开场，skill 自动拉起前后端"
   - 30 页还原 → "8 路子代理并行镜像样板"
   - 18 表建库 → "契约收口在建库前扫雷"
   - 联调打穿 → "真数据第一次流动的时刻"
3. **大数字页**：65,000+ / 0 手写 / 30 页 / 74+ 端点
4. **配方化断崖图**：首模块投入 10 → 第二模块 2（一次编译通过）——"首模块的投入不是成本，是买断后续所有模块的配方费"
5. **诚实账**（1 页）：返工占早期开销一半 + 金句"AI 提供马力，门禁提供刹车，人掌方向盘"

### 工单系统（10 min，对应 10.2）

1. **三天 timeline**：Day1 三板斧 → Day2 六角色走查 → Day3 并行修复
2. 重点讲一个漏洞的故事：JWT 弱默认密钥——长期存在、无人发现，AI 一次走查挖出，还逐一核实了"所有 yml/环境变量零注入"这种人不会去穷举的证据链
3. 诚实汇报口径（也是给听众做示范）：**3 个 P0 修掉 2 个，JWT 弱密钥当时仍未修**——"我们连汇报都要求带证据、列未尽事项"

**过渡句**："以上都是别人的项目。接下来一分钟，是你们每个人明天就能用的。"

## 第 ④ 幕：现场 Demo（5 min，对应 4.4 🎬）

**演示**：`/daily-report` —— 扫 git 提交自动生成日报，人工确认后上报。

**演示脚本**：
1. 打开今天真实在用的项目终端（提前确认今日有提交记录）
2. 敲 `/daily-report`，等待生成期间讲原理："它打的不是'写'，是'回忆'——git 记录本来就是你最准确的工作日志"
3. 日报出来，念一遍，点确认
4. 收口："每天省 14 分钟，一个月就是接近一个工作日。**只这一条命令，订阅费就回本了**——后面所有提效都是净赚"

**失败预案**：提前录好一段 40 秒的演示录屏 / 截图序列，命令卡住超过 30 秒立即切换："网络不给面子，看录屏版"——不要现场调试。
